## Abhaengigkeiten und Zusammenspiel

pcloud-commander ist bewusst schlank und nutzt bestehende Komponenten aus deinem Stack:

| Komponente | Zweck | Repository |
|---|---|---|
| pcloud-tools | Binary API Library (`pcloud_bin_lib.py`) und Upload-/Sync-Skripte | https://github.com/lastphoenx/pcloud-tools |
| script-manager-ui | Externer Script-Katalog (`scripts.yaml`) fuer Dashboard/Formulare | https://github.com/lastphoenx/script-manager-ui |

### Woher pcloud-commander diese Dateien laedt

- `pcloud_bin_lib.py` wird u.a. unter `/opt/apps/pcloud-tools/main/pcloud_bin_lib.py` gesucht.
- `scripts.yaml` wird u.a. unter `/opt/apps/script-manager-ui/scripts.yaml` gesucht.
- `.env` wird bevorzugt aus `pcloud-tools` geladen (u.a. `/opt/apps/pcloud-tools/main/.env`).

## Features (aktueller Stand)

- Zwei-Pane-Ansicht: lokal (links) und pCloud (rechts)
- Dateimetadaten in der UI (Groesse, mtime)
- Dateiaktionen je Pane (`a`): copy/move/delete/rename/new folder
- Script-Dashboard (`s`) mit Param-Formularen aus `scripts.yaml`
- Browse-Buttons fuer lokale und pCloud-Pfade in Script-Parametern
- High-Risk-Script-Schutz mit sicheren Defaults und expliziter Bestaetigung
- Quick-Upload/Sync Wizard mit Schutz vor unbeabsichtigtem Root-Upload

## Installation

Diese Anleitung orientiert sich am aktuellen Zielzustand unter `/opt/apps`.

### 1. Repositories bereitstellen

Beispielstruktur:

- `/opt/apps/pcloud-commander/main`
- `/opt/apps/pcloud-tools/main`
- `/opt/apps/script-manager-ui`

### 2. Python-Umgebung

Der mitgelieferte Starter `pcloud-commander.sh` nutzt bewusst das venv von `pcloud-tools`:

- `VENV_PATH=/opt/apps/pcloud-tools/venv`

Stelle sicher, dass dort mindestens `textual` und `pyyaml` verfuegbar sind.

### 3. pcloud-tools Konfiguration

Eine funktionierende `.env` in `pcloud-tools` ist erforderlich, da die API-Konfiguration von dort geladen wird.

### 4. script-manager-ui Katalog

Wenn das Script-Dashboard genutzt werden soll, muss `scripts.yaml` unter einem der Suchpfade liegen (empfohlen: `/opt/apps/script-manager-ui/scripts.yaml`).

## Starten

Empfohlener Start wie von dir genutzt:

```bash
sudo /opt/apps/pcloud-commander/main/pcloud-commander.sh
```

Optional mit CLI-Parametern:

```bash
sudo /opt/apps/pcloud-commander/main/pcloud-commander.sh --theme ocean --local-root /srv --download-dir /srv/nas/restore
```

## Bedienung (Kurzuebersicht)

- `Tab`: Pane wechseln
- `a`: Dateiaktionen im aktiven Pane
- `s`: Script-Dashboard / Quick Actions
- `d`: Download (pCloud -> lokal)
- `F8`: Loeschen im aktiven pCloud-Kontext
- `r`: Refresh
- `u` oder `Backspace`: pCloud eine Ebene hoch
- `q`: Beenden
